Secure Automation & Creation

Building deterministic document import workflows that automated boundary and assessment setup while prioritizing validation, predictable behavior, user review, and security-minded development.

Overview

My Role

One of the company’s goals was to reduce the time required to prepare boundaries and cybersecurity assessments. Auditors and assessors often needed to copy and compile information manually from existing documents into multiple application fields, making the process repetitive and time-consuming.

I designed and developed a document import workflow that extracted known information from uploaded files, mapped it into application fields, and allowed users to review and complete the data before it became part of the system.

Role: Software Developer Workflow Design Python Development Front-End Integration SQL Integration Validation

Problem

The Challenge

Creating a new boundary or assessment required users to locate information across multiple source documents and manually enter it into the application. A process intended to simplify cybersecurity and compliance work still required significant administrative effort before the user could begin the actual assessment.

The import process also needed to behave predictably. Automatically accepting uncertain or incorrectly mapped information could create inaccurate application records and reduce user trust. An AI-dependent workflow could also introduce additional approval, explainability, privacy, or deployment concerns for some enterprise and government environments.

The challenge was to automate the repetitive parts of the process while preserving user control, validation, and clear visibility into what information would be stored.

Contributions

Design & Development

My work covered the extraction logic, interface workflow, field mapping, validation process, and integration of imported data with the application’s existing boundary and assessment functionality.

  • Built Python-based document extractors for supported Word, Excel, PDF, and CSV files.
  • Developed mapping logic that connected recognized document content to boundary and assessment fields.
  • Integrated document import into the application’s existing creation workflow rather than requiring users to move through a separate disconnected tool.
  • Created a draft-and-review process that allowed users to inspect mapped information before completing an import.
  • Allowed users to manually complete missing or unrecognized fields before committing data to the application.
  • Connected imported information to SQL-backed boundary and assessment workflows.
  • Added validation and user-confirmation steps so imported information was not treated as trusted application data automatically.
  • Designed the workflow to reduce dependence on probabilistic AI processing while still automating repetitive document-handling tasks.

Approach

Integrated Creation and Verification Workflow

I initially explored the import feature as a dedicated Import Center. As the workflow developed, I recognized that users would benefit from importing information at the same point where they created a boundary or assessment.

I integrated the import process into the application’s existing creation wizard. Users could upload a document, review extracted and mapped fields, manually provide missing information, and verify the completed draft before anything was saved.

This approach followed a verify-before-trust model. The system assisted the user by extracting and organizing information, but the user remained responsible for reviewing the result before the application accepted it as final data.

Result

A Faster Workflow with User-controlled Validation

The final workflow reduced repetitive data entry by extracting and organizing information that users would otherwise need to copy manually across multiple application fields.

Integrating the feature into the creation wizard kept the process connected to the user’s existing workflow, while the draft, review, and confirmation stages helped prevent uncertain imported information from being accepted automatically.

The project gave me practical experience with Python-based file processing, document extraction, field mapping, SQL-backed workflows, front-end integration, validation, and security-minded automation within an enterprise application.
